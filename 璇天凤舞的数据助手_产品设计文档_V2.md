# 璇天凤舞的数据助手 - 产品设计文档

## 1. 产品定位

**核心定位**：面向有二级市场投资经验或兴趣的用户的专业工具类小程序

**价值主张**：提供从交易数据获取渠道到交易策略介绍、到交易辅助工具推荐的一站式服务平台，帮助用户做出更明智的投资决策。

**目标用户**：

- 有一定投资经验的二级市场投资者
- 对量化投资感兴趣的用户
- 需要专业工具辅助交易的投资者

## 2. 整体架构

### 2.1 页面结构

| 页面   | 核心功能                    | 文件路径 |
| ---- | ----------------------- | -------- |
| 主页   | 产品定位介绍、功能板块导航、使用指南     | `pages/index/index` |
| 交易数据 | 投资标的数据获取渠道介绍、数据来源、官网链接  | `pages/data/data` |
| 交易策略 | 交易策略分类介绍、策略社区、公众号、研究报告 | `pages/strategy/strategy` |
| 交易工具 | 交易软件、分析软件推荐、资讯/回测/交易类工具 | `pages/tools/tools` |
| 通用列表页 | 各分类的完整列表展示（点击"查看全部"跳转） | `pages/common/list` |
| 详情页 | 各项目的详细内容展示 | `pages/detail/detail` |

### 2.2 技术架构

- **前端**：原生微信小程序
- **数据来源**：
  - **首页静态数据**：各标签页首页内容写死在页面JS文件中
  - **列表页动态数据**：从腾讯云COS加载 `categories.json` 文件
  - **详情页动态数据**：从腾讯云COS加载对应 `details/{id}.json` 文件
- **数据存储**：腾讯云COS（对象存储）
- **更新机制**：修改COS上的JSON文件即可更新内容，无需重新发布小程序

### 2.3 数据文件结构

```
data/                              # 数据文件夹（上传至COS）
├── trading_data/                  # 交易数据模块
│   ├── categories.json            # 分类配置（含列表展示数据）
│   └── details/                   # 详情页数据
│       ├── tushare.json
│       ├── akshare.json
│       ├── eastmoney_data.json
│       └── ...
├── trading_strategy/              # 交易策略模块
│   ├── categories.json            # 分类配置
│   └── details/                   # 详情页数据
│       ├── joinquant.json
│       ├── citics.json
│       └── ...
├── trading_tools/                 # 交易工具模块
│   ├── categories.json            # 分类配置
│   └── details/                   # 详情页数据
│       ├── eastmoney.json
│       ├── tonghuashun.json
│       └── ...
└── shangzheng_index.csv           # 上证指数数据（K线图使用）
```

### 2.4 页面跳转逻辑

```
首页（静态数据）
    ↓ 点击各板块入口
标签页首页（静态数据）
    ↓ 点击"查看全部"
通用列表页（加载 categories.json）
    ↓ 点击"查看详情"
详情页（加载 details/{id}.json）
```

### 2.5 数据更新流程

1. **修改数据文件**：在本地修改 `data/` 文件夹下的JSON文件
2. **上传至COS**：将修改后的文件上传到腾讯云COS对应路径
3. **自动生效**：用户下次访问时自动获取最新数据（依赖HTTP缓存策略）

**注意**：首次加载或缓存过期后会重新获取最新数据。

## 3. 页面详细设计

### 3.1 主页（首页）

**页面结构**：

1. **头部区域**：产品名称、Slogan
2. **核心功能介绍**：三大功能板块的简要介绍（交易数据、交易策略、交易辅助工具）
3. **使用指南**：简要说明如何使用小程序
4. **底部导航**：四个标签页导航

**设计风格**：

- 专业金融风格，深蓝+白色为主色调
- 简洁大气，数据可视化元素
- 响应式布局，适配不同设备

### 3.2 交易数据（数据中心）

**页面结构**：

1. **数据分类**：股票（A股/港股/美股/历史数据）、期货（商品/股指/国债）、数字货币、宏观经济
2. **免费数据源网站**：
   - 股票：Tushare、AKShare、东方财富数据中心、Yahoo Finance、网易财经历史数据
   - 期货：和讯期货、博易大师数据页面、郑商所/大商所官网
   - 数字货币：CCXT、CoinGecko、CoinMarketCap
   - 宏观经济：国家统计局、中国人民银行、FRED、Wind（公开部分）
3. **内容详情**：每个数据源的名称、主要数据类型、官网链接、简介、详情页面

**功能特性**：

- 数据分类展示（二级分类）
- 一键跳转官网
- 数据类型说明
- 推荐数据来源

### 3.3 交易策略（策略广场）

**页面结构**：

1. **策略分类**：股票、期货、数字货币（横向滑动切换）
2. **内容板块**（每个分类下）：
   - 策略社区：聚宽社区、果仁网、掘金量化社区等
   - 推荐公众号：交易员说、人人宽客、BigQuant量化等
   - 研究报告：相关研究报告链接
3. **策略详情**：策略名称、核心逻辑、适用场景、学习链接

**功能特性**：

- 策略分类浏览
- 详细的策略介绍
- 学习资源链接
- 策略适用场景说明

### 3.4 交易辅助工具（工具中心）

**页面结构**：

1. **工具分类**：回测与量化框架、数据获取与清洗工具、盯盘与提醒工具、订单管理与风控、看盘与图表分析
2. **工具介绍**：
   - 回测框架：Backtrader、VectorBT、Zipline、VN.PY、Pine Script（TradingView）
   - 数据与API工具：CCXT（加密货币）、AkShare、Pandas
   - 看盘与提醒：TradingView、自选股/同花顺
3. **内容详情**：每个工具的名称、核心功能、官网/GitHub链接、简介

**功能特性**：

- 工具分类展示
- 一键跳转官网/GitHub

| updateTime | string | 否 | 更新时间 |

### 4.4 通用列表页参数说明

列表页通过URL参数获取数据：

| 参数 | 说明 | 可选值 |
|------|------|--------|
| module | 模块名称 | `trading_data`, `trading_strategy`, `trading_tools` |
| category | 一级分类 | `stock`, `futures`, `crypto`, `macro`（交易数据） |
| subtype | 二级分类 | `items`（交易数据）, `communities`, `officialAccounts`, `research`（交易策略）, `news`, `backtest`, `trading`（交易工具） |

**示例URL**：
```
/pages/common/list?module=trading_strategy&category=stock&subtype=research
```

### 4.5 详情页参数说明

详情页通过URL参数获取数据：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| module | 模块名称 | `trading_data` |
| id | 项目ID | `tushare` |

**示例URL**：
```
/pages/detail/detail?module=trading_data&id=tushare
```
